# Dishly  
A Global Recipe Index Web Application

Dishly is a responsive web application built with **Next.js** that functions as an **accessible online index of dishes from around the world**. The application fetches and displays JSON data from a public REST API and is deployed online for public access.

---

## Test Case Mapping (TC1–TC10)

**TC1 – Responsive layout on desktop**  
Dishly uses responsive layouts to ensure proper display and usability on desktop screens.

**TC2 – Responsive layout on mobile**  
The interface adapts to mobile screen sizes using responsive design techniques, ensuring readability and functional interaction on smaller devices.

**TC3 – Successfully fetches JSON data from REST API**  
The application fetches live JSON data from the **TheMealDB REST API** using client-side data fetching.

**TC4 – Displays fetched data correctly**  
Fetched recipe data is dynamically rendered into recipe cards and detail views, including images, ingredients, and instructions.

**TC5 – Proper use of CSS/JS frameworks for styling**  
Dishly is built with **Next.js** and styled using **Tailwind CSS combined with custom CSS**, demonstrating proper framework usage.

**TC6 – Interactive elements work correctly**  
Interactive features include searchable recipe lists, filters by cuisine area, category, and ingredient, clickable recipe cards, pagination, and embedded videos.

**TC7 – Application is deployed and accessible online**  
The web application is deployed using **Vercel** and is publicly accessible via a live URL.

**TC8 – Professional and clean design**  
The application follows a clean visual hierarchy, consistent typography, and spacing, resulting in a professional appearance.

**TC9 – Overall usability and user experience**  
Dishly provides clear navigation, readable content, loading and error states, and an intuitive browsing experience.
